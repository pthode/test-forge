## Detailed findings by category

### Naming

No new issues. All identifiers use full words (`_capacity`, `_entries`, `_hits`, `_misses`, `_evictions`, `hits`, `misses`, `evictions`, `size`). The `LruCache` vs `LRUCache` divergence from CONSTITUTION §3 is already tracked as B-008.

### Cohesion

Each method has a single, clearly nameable responsibility. `get` looks up and promotes; `put` stores and evicts if needed; `stats` snapshots counters. `__init__` validates and initialises — inseparable concerns. `CacheStats` is the return type of `stats()` and lives in the same module, which is the right level of proximity. No cohesion issues.

### Duplication

`len(self._entries)` appears at lines 114 and 131 in different contexts (capacity guard vs. stat snapshot). No duplication concern — extracting a helper would add indirection without payoff. No other duplicated logic detected.

### Abstraction quality

`CacheStats` as a `@dataclass(frozen=True)` is exactly right: attribute access, immutable snapshot semantics, clear type name in tracebacks, stdlib-only. The internal `OrderedDict` typed as `OrderedDict[Hashable, Any]` is the most precise annotation available without generics (B-013 tracks the generic upgrade path). The get hit path's three sequential operations on `_entries` (`__contains__`, `move_to_end`, `__getitem__`) are a candidate for a private helper, but the helper would be called from exactly one site. No issue. Double-hash-lookup is tracked as B-009.

### Pattern consistency

`duration.py` establishes a module docstring style: single-sentence, verb-first imperative. `cache.py:1-6` opens with a noun-phrase. This diverges from the house style. Already tracked as B-014.

No other pattern divergences found. Both modules use no `__all__`, use module-private names with leading underscores, use f-strings in error messages, and use only stdlib imports.
